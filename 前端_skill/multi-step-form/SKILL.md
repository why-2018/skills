# Multi Step Form

## Quick Start

处理分步表单时：

1. 先查项目已有步骤条、表单校验、草稿保存、路由离开确认和提交接口写法。
2. 明确每一步的字段、校验边界、是否可跳步、是否保存草稿和最终提交结构。
3. 每一步只负责本步骤交互，跨步骤数据由页面层或 Store 管理。
4. 复杂流程先保证线性成功路径，再补跳转、回退、草稿和异常处理。

## Must Do

- 下一步前必须校验当前步骤，不能把错误延迟到最终提交才暴露。
- 上一步不能丢失已填写数据，也不能触发不必要的提交。
- 草稿保存要区分自动保存、手动保存和最终提交。
- 步骤状态要能表达未开始、进行中、已完成、错误和禁用。
- 最终提交前要合并并转换所有步骤数据，避免字段命名冲突。
- 路由离开或关闭弹窗时要处理未保存变更确认。

## Verification

- 检查下一步、上一步、跳步、草稿、最终提交和提交失败。
- 检查刷新页面、接口回显、校验错误定位和离开确认。
- 运行项目约定的类型检查或构建命令。
