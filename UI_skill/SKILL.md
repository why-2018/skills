## 1. 使用边界与输出要求
### 1.1 适用场景
- 适用于 **B 端后台 / 管理台 / 数据看板 / 配置页 / 列表页 / 详情页 / 表单页 / 弹窗与抽屉** 等桌面 Web 场景。
- 适用于需要同时约束 **视觉规范、交互状态、前端实现方式、验收标准** 的任务。
- 适用于以下工作类型：
  - 页面/模块 UI 设计
  - 设计稿转前端实现
  - 现有页面视觉重构
  - 组件规范补齐
  - 设计或代码评审

### 1.2 不适用场景
- 不适用于品牌官网、营销活动页、故事化落地页、强视觉概念页。
- 不适用于移动端 App、小程序原生界面、游戏化界面。
- 若项目已有更高优先级的品牌规范、设计系统或组件库约束，应 **优先继承现有约束**，本规范作为补充，而不是覆盖已有体系。

### 1.3 默认执行方式
- 接到任务后，优先识别页面类型、核心任务路径、主要信息层级与关键组件。
- 优先保证 **布局清晰、状态完整、组件复用、Token 落地、可访问性达标**，其次再做视觉细节强化。
- 对于列表、表单、弹窗、权限、加载、空态、错误态等高频 B 端场景，必须主动补齐缺失状态，而不是只输出静态正常态。

### 1.4 默认输出要求
- 如用户要求设计方案：输出页面结构、模块分区、关键交互、状态覆盖、视觉与组件建议。
- 如用户要求前端实现：输出或直接实现符合本规范的结构、样式、状态与 Token 映射，禁止只给高层空泛建议。
- 如用户要求评审：优先指出与本规范冲突的布局、层级、状态、可访问性、Token 使用问题，并给出修正方向。

## 2. 设计原则
- **清晰高效**：通过明确的视觉层级与极简信息架构，降低用户认知负荷；精简操作路径，确保核心任务可在最短路径内完成，提升操作效率。
- **高度一致**：视觉元素、交互模式、反馈方式及术语使用在产品内保持统一，遵循用户心智模型，降低学习成本，增强产品专业感与可信度。
- **即时反馈**：每个用户操作后立即呈现明确的状态反馈（如加载中、成功、失败等），消除不确定感，帮助用户随时理解系统响应与当前状态。
- **灵活可扩展**：采用响应式布局与模块化设计，完美适配 1280~1920 等主流分辨率；预留业务扩展空间，支持功能模块的动态增减与未来迭代。


## 3. 布局规范
### 3.1 页面结构
- **上下布局**：顶部导航(56px) + 侧边菜单(208px) + 内容区域。
- **左右布局**：侧边菜单(208px) + 内容区域（顶部可含面包屑/操作栏）。
- **适配分辨率**：最佳设计宽度**1440px**，内容区域左右边距16px，最大内容宽度1200px。支持1920、1440、1366、1280分辨率，低于1280出现横向滚动。
- **层级**：背景层(0) → 内容层(1) → 全局控制层(10) → 内容弹层(1000)，自下而上。

### 3.2 栅格系统
- **栅格数量**：默认使用 **24栅格**，基于Flex或Grid实现。响应式场景可调整为20/16栅格。
- **栅格间距(Gutter)**：
  - 卡片列表：`12px`
  - 紧凑列表：`8px`
- **页面边距(Margin)**：内容区域上下左右边距均为 `16px`。
- **元素间距**：必须使用以下偶数序列：`2px`， `4px`， `8px`， `12px`， `16px`， `20px`。
  - 同组元素间：`4px` 或 `8px`
  - 不同功能组之间：`12px`
  - 区块之间：`12px` 


## 4. 色彩规范
所有颜色必须定义为CSS变量（Design Tokens），禁止硬编码。
- 以下色值为 **Token 源值** 或设计基准值，可用于定义全局变量；在组件与页面样式中应优先引用 Token，避免直接写入十六进制色值。

### 4.1 主色
- **主色**：`#2664FF`（应用于主要按钮、链接、选中态、进度条）
- **触摸状态**：`#6792FF`
- **点击状态**：`#1A4ECF`

### 4.2 辅助色
- **成功**：`#39D88B`（成功提示、完成状态）
- **警告**：`#FF7E25`（警告提示、待处理状态）
- **错误**：`#FF624E`（错误提示、删除操作）

### 4.3 中性色
| 用途               | 色值       | 应用场景                         |
|--------------------|------------|----------------------------------|
| 标题/正文          | `#1E232C`  | 页面标题、表头、重要文本         |
| 次级文本           | `#434B5C`  | 正文内容、灰底按钮文字           |
| 次次级文本         | `#8792A8`  | 辅助信息、次要元素               |
| 提示说明文字       | `#A1AABE`  | 占位符、说明文字                 |
| 占位/禁用          | `#8792A8`  | 禁用状态文字                     |
| 分割线             | `#E9ECF2`  | 边框、分割线                     |
| 背景色（内容区）   | `#F6F7F9`  | 页面背景、卡片背景               |
| 组件背景           | `#FFFFFF`  | 输入框、卡片、弹窗背景           |

- **使用限制**：主色在单个界面中的使用面积不得超过30%。辅助色仅用于对应的状态提示。


## 5. 字体与排版
### 5.1 字体家族
- 默认：`-apple-system, BlinkMacSystemFont, 'Segoe UI', Roboto, 'Helvetica Neue', Arial, sans-serif`
- 数字字体：`DIN` 或 `Monospaced`（表格数字、统计卡片）

### 5.2 字号与行高
`【必须】` 全局字号严格限制在以下5个层级内，禁止使用其他字号。

| 类型         | 字号 (px) | 行高 (px) | 使用场景                     |
|--------------|-----------|-----------|------------------------------|
| 大标题       | 24        | 32        | 卡片数字、数据总览           |
| 主标题       | 20        | 28        | 页面唯一标题                 |
| 小标题       | 16        | 24        | 卡片标题、模块标题           |
| 正文（默认）  | 14        | 22        | 正文、表单标签、按钮文字     |
| 辅助文本     | 12        | 20        | 提示、标签、说明文字         |
### 5.2.1 排版例外位
- 允许 `18px` 仅用于顶部品牌标题（如产品名称/系统名）。
- `10px` 仅允许用于图标装饰或极小辅助图形，不得用于正文、标题、按钮、表单、表格文本。

### 5.3 字重
- 正文/辅助：`400` (Normal)
- 小标题/表头/大标题/主标题：`500` (Medium)
- 关键数据：`600` (Semi-bold)

### 5.4 段落间距
- 标题与内容间距：`12px`
- 段落之间间距：`8px`
- 列表项之间间距：`8px`


## 6. 图标
- **格式**：优先 SVG；允许 RemixIcon 字体图标（过渡阶段），并给出迁移策略（新组件优先 SVG）。
- **尺寸**：按钮内16x16，导航20x20，空状态24x24，侧边菜单16x16。
- **颜色**：默认继承文本色，侧边菜单图标固定 `#9CA6BC`，可自定义状态色（如悬停变主色）。
- **图标库**：优先使用 [RemixIcon](https://remixicon.com/)，定制图标需对齐风格。
- **对齐**：图标与文字垂直居中对齐，间距8px；纯图标按钮热区不小于32x32。


## 7. 按钮
### 7.1 类型
| 类型     | 样式                                      | 悬停效果               | 点击效果               |
|----------|-------------------------------------------|------------------------|------------------------|
| 主要按钮 | 背景 `#2664FF`，白色文字                  | 背景 `#6792FF`         | 背景 `#1A4ECF`         |
| 默认按钮 | 背景 `#F6F7F9`，文字 `#434B5C`            | 背景 `#EDF1F6`         | 背景 `#E0E5ED`         |
| 虚线按钮 | 边框虚线 `#D0D7E2`，文字 `#434B5C`        | 边框变主色             | 背景 `#F6F7F9`         |
| 文本按钮 | 无边框，文字 `#434B5C`                    | 背景 `#F6F7F9`         | 背景 `#EDF1F6`         |
| 链接按钮 | 文字链接样式，下划线                       | 颜色变主色             | 颜色变深               |
| 禁用按钮 | 背景 `#F6F7F9`，文字 `#8792A8`            | 无变化                 | 无变化                 |

### 7.2 尺寸
| 尺寸   | 高度 | 内边距(左右) | 字号 | 图标尺寸 | 适用场景           |
|--------|------|--------------|------|----------|--------------------|
| large  | 40px | 16px         | 16px | 18x18    | 页面级主要操作     |
| middle | 36px | 12px         | 14px | 16x16    | 默认按钮、表格操作 |
| small  | 24px | 8px          | 12px | 12x12    | 紧凑场景、标签内   |

### 7.3 状态
- 正常、悬停、点击、禁用（`opacity: 0.5`，不可点击，无阴影）
- 按钮内图标与文字间距统一为4px
- 按钮组之间间距12px（水平）或8px（垂直）


## 8. 表单
### 8.1 输入框
- **尺寸**：高度 `36px`，宽度自适应，最小宽度 `192px`。
- **背景**：`#F6F7F9`，圆角 `8px`，无边框。
- **内边距**：左右 `12px`，上下 `8px`。
- **聚焦态**：出现 `1px` 主色边框 `#2664FF`，无阴影。
- **禁用态**：背景 `#F6F7F9`，文字 `#8792A8`，不可点击。
- **错误态**：边框 `#FF624E`，下方显示红色提示文字（字号12px，颜色 `#FF624E`）。

### 8.2 表单项布局
- **水平布局**：标签右对齐，输入框宽度 `320px`，标签与输入框间距 `4px`。
- **垂直布局**：标签在上，标签与输入框间距 `8px`，输入框宽度 `320px`。
- **表单项间距**：上下间距 `16px`，分组间距 `24px`。

### 8.3 选择器/日期等
- 高度、背景、圆角与输入框一致，下拉箭头图标 `#8792A8`。
- 下拉菜单：背景白色，边框 `#E9ECF2`，圆角8px，最小宽度120px，最大高度256px（可滚动），选项高度36px，悬停背景 `#F6F7F9`。

### 8.4 筛选栏（表格上方查询区）
- **布局结构**  
  父容器采用 Flex 布局：`display: flex; flex-wrap: wrap; align-items: flex-start; gap: 16px;`  
  - 筛选框容器：`flex: 1; display: flex; flex-wrap: wrap; gap: 16px;`  
  - 按钮组：宽度固定 **156px～160px**，包含“查询”和“重置”两个按钮，按钮间距 **8px**。
- **筛选框宽度**  
  每个筛选框固定宽度 **196px**（`flex: 0 0 196px;`），一行最多显示 5 个；超过 5 个时自动换行，换行后保持 196px 且左对齐，确保多行垂直对齐。
- **间距控制**  
  所有间距统一为 **16px**（筛选框之间、筛选框与按钮组之间），按钮组内按钮间距 **8px**。
- **按钮组跟随**  
  按钮组与筛选框容器在父容器中自动排列：空间充足时与筛选框同行右侧；空间不足时自动换行至下一行，始终与筛选框容器保持 16px 间距。
- **响应式适配**  
  根据容器宽度自动调整每行筛选框数量：  
  - ≥1156px → 5 个  
  - 960px ～ 1155px → 4 个  
  - 764px ～ 959px → 3 个  
  - 568px ～ 763px → 2 个  
  - <568px → 1 个
- **样式**  
  筛选框遵循 **8.1 输入框** 规范（高度 36px、背景 `#F6F7F9`、圆角 8px）；按钮遵循 **7.2 按钮** 规范（默认按钮尺寸 middle，36px 高）。

### 8.5 表单校验规范
- **校验触发时机**：
  - `blur`：格式合法性（如邮箱、手机号、日期格式）
  - `change`：强约束字段（如下拉必选、范围联动）
  - `submit`：最终完整校验（阻止无效提交）
- **错误提示位置**：错误文案显示在字段下方，左对齐，颜色 `#FF624E`，字号 `12px`，行高 `20px`。
- **首错定位**：提交失败时自动滚动并聚焦到第一个错误字段。
- **提交控制**：提交按钮在请求期间进入 `loading` 且不可重复点击；请求完成后恢复。

## 9. 表格
### 9.1 基础样式
- **边框**：全边框，颜色 `#E9ECF2`，线宽1px。
- **行高**：默认48px，紧凑44px（用于数据密集场景）。
- **表头**：背景 `#F6F7F9`，字号14px，字重500，文字 `#434B5C`，左对齐。
- **内容**：字号14px，文字 `#1E232C`。
- **隔行变色**：可选，偶数行背景 `#F6F7F9`（与表头一致）。
- **悬停行**：背景 `#F6F7F9`（与表头一致）。
- **列分割**：每列右侧1px `#E9ECF2`，最后一列无分割。
- **列宽分配**：除操作列外，其余列根据内容比例平均分配剩余宽度（即“平分”），支持列宽拖动自定义。
- **紧凑模式例外**：表格启用紧凑模式时，允许 `12px` 字号 + `44px` 行高；默认模式保持 `14px` 字号 + `48px` 行高。

### 9.2 操作列
- 操作按钮使用文本按钮或图标按钮，间距 `12px`。

### 9.3 分页
- 位置：表格右下角，上下边距16px。
- 分页按钮：尺寸36x36，背景 `#F6F7F9`，无边框，圆角8px，文字 `#434B5C`，悬停背景 `#EDF1F6`。
- 当前页：背景 `#2664FF`，文字白色。
- 每页条数选择：10、20、50、100，下拉选择器高度36px。

### 9.4 列规范（B端列表增强）
- **文本截断**：单元格默认单行省略（ellipsis）；超长内容通过 Tooltip 展示完整值。
- **对齐规则**：文本左对齐，数值右对齐，状态/标签居中或左对齐保持一致。
- **时间格式**：统一 `YYYY-MM-DD HH:mm:ss`，无值显示 `--`。
- **操作列**：最小宽度建议 `120px`；按钮间距建议 `8px~12px`；危险操作使用错误色并二次确认。
- **批量操作**：仅在选择行数 > 0 时激活批量按钮，并展示已选数量。


## 10. 导航
### 10.1 侧边菜单
- 宽度：展开208px，折叠64px。
- 背景：白色 `#FFFFFF`，无边框。
- 菜单项高度40px，内边距24px（展开时），折叠时居中图标。
- 图标：16x16，颜色 `#9CA6BC`，展开时右侧留白12px。
- 文字：14px，颜色 `#434B5C`。
- 选中项：背景 `#2664FF`（或高亮条），文字白色，图标白色。
- 悬停项：背景 `#F6F7F9`。

### 10.2 顶部导航
- 高度56px，背景白色，下边框1px `#E9ECF2`。
- 面包屑：14px，分隔符“/”，颜色 `#8792A8`，最后一项 `#1E232C`。
- 右侧操作区：用户头像、通知图标等，间距16px。


## 11. 反馈组件
### 11.1 消息提示
- 位置：顶部居中，距离顶部16px，水平居中，z-index 1100。
- 宽度：自适应，最大400px，内边距12px 24px。
- 类型：成功（`#39D88B`图标）、错误（`#FF624E`）、警告（`#FF7E25`）、信息（`#2664FF`）。
- 自动关闭：成功/信息/警告3秒，错误需手动关闭（有关闭图标）。

### 11.2 对话框
- `【必须】` 尺寸规范：
  - 小型弹窗：400px（确认框、简单提示）
  - 中型弹窗：560px～800px（表单填写）
  - 大型抽屉：40%～60% 视口宽度（复杂数据展示）
- 圆角：20px，背景白色。
- 标题高度54px，内边距24px，字号16px，字重500，分割线 `#E9ECF2`。
- 底部按钮右对齐，内边距24px，按钮间距12px。
- 遮罩背景：`rgba(30,35,44,0.30)`，z-index 1000。

### 11.3 加载中
- 局部加载：Spin组件，居中显示，遮罩半透明（`rgba(255,255,255,0.6)`），z-index 10。
- 全屏加载：遮罩 `rgba(0,0,0,0.3)`，禁用页面操作，z-index 2000。

### 11.4 页面状态（列表/表单通用）
- **空态**：当无数据时显示空态图标、说明文案与建议操作（如“新建”/“去配置”）。
- **错误态**：请求失败时显示错误提示、错误原因（可选）与“重试”操作。
- **无权限态**：显示权限说明与申请入口（或联系管理员路径），不展示可执行高权限按钮。
- **初始化加载态**：首屏加载优先使用骨架屏或区域级 loading，避免页面闪烁。
- **局部刷新态**：筛选、分页、局部操作使用区域 loading，避免全屏遮罩阻断。

### 11.5 权限态规范
- **按钮权限**：无权限按钮采用禁用或隐藏策略，页面内保持一致，不可混用。
- **字段权限**：只读场景明确展示不可编辑态（只读样式与提示文案）。
- **操作权限**：行级操作按权限裁剪，危险操作权限单独校验。
- **提示文案**：权限不足提示使用统一文案模板，说明“缺少什么权限”和“如何申请”。


## 12. 其他组件
### 12.1 卡片
- 圆角 `8px`，边框 `1px solid #E9ECF2`，内边距 `20px`。
- 阴影可选：`0 2px 8px rgba(0,0,0,0.08)`。
- 卡片之间间距16px。

### 12.2 标签/Tag
- 圆角4px，内边距4px 8px，字号12px/20px，字重400。
- 背景 `#F6F7F9`，文字 `#434B5C`。
- 可关闭标签：右侧关闭图标，间距4px，悬停背景 `#EDF1F6`。

### 12.3 步骤条
- 水平步骤，间距120px（可根据内容调整）。
- 图标尺寸24px，已完成步骤图标 `#39D88B`，当前步骤图标 `#2664FF`，待处理 `#8792A8`。
- 连接线：1px solid `#E9ECF2`，长度与间距匹配。


## 13. 响应式与适配
- **断点**：
  - `<1200px`：侧边菜单自动折叠，内容区出现横向滚动。
  - `<992px`：表格改为卡片式列表（移动端适配，B端后台可选）。
- **自适应**：栅格列宽百分比，侧边菜单固定宽度。
- **字体**：所有字号使用px，不随视口缩放。


## 14. 动效
- **过渡**：`0.3s cubic-bezier(0.645, 0.045, 0.355, 1)`（悬停、展开/折叠）。
- **弹窗出现**：缩放+淡入（0.2s），遮罩淡入（0.1s）。
- **按钮点击**：轻微缩放（0.95→1）或背景变化，禁用过度动画。
- **避免**：同时过多元素动画，重要操作无干扰。


## 15. 代码实现要求
1.  **设计令牌**：所有颜色、字体、间距、圆角等必须定义为CSS变量（例如 `--color-primary`， `--spacing-md`），并在全局根元素中声明。
2.  **组件库**：推荐使用Ant Design (React) 或 Element Plus (Vue) 作为基础组件库，并严格按照本规范覆盖其主题变量。
3.  **复用优先**：若项目已存在设计系统、Token 文件、主题配置、基础组件封装，必须优先复用，避免重复发明新的视觉体系或平行组件。
4.  **状态分层要求**：
   - 关键流程组件（按钮/表单/表格/弹窗）必须具备 `normal`、`hover`、`active`、`disabled` 四种基础状态。
   - 涉及异步请求的操作必须具备 `loading` 状态（按钮级或区域级）。
   - 成功/失败反馈统一通过页面级消息组件实现（如 Message/Toast/Alert），禁止仅依赖浏览器原生 `alert/confirm`。
5.  **命名规范**：自定义样式文件必须使用 **BEM** (Block Element Modifier) 命名规范，并按组件拆分。
6.  **实现策略**：优先通过主题变量、语义类名、组件封装与状态类实现规范，不建议通过大面积行内样式或散乱覆盖实现。
7.  **浏览器兼容性**：必须兼容Chrome ≥ 80， Firefox ≥ 75， Edge ≥ 80。

### 15.1 Token 命名字典（建议）
- **颜色**：`--color-primary`，`--color-success`，`--color-warning`，`--color-danger`，`--color-text-*`，`--color-bg-*`，`--color-border-*`
- **字体**：`--font-size-*`，`--line-height-*`，`--font-weight-*`
- **间距**：`--space-2`，`--space-4`，`--space-8`，`--space-12`，`--space-16`，`--space-20`
- **圆角**：`--radius-4`，`--radius-8`，`--radius-20`
- **层级**：`--z-base`，`--z-sticky`，`--z-dropdown`，`--z-modal`，`--z-toast`
- **动效**：`--motion-duration-fast`，`--motion-duration-base`，`--motion-ease-standard`

## 16. 可访问性最小标准
- **键盘可达**：主要交互（按钮、输入、分页、菜单、弹窗操作）必须可通过键盘访问。
- **焦点可见**：可交互元素必须有清晰 focus 样式，不得移除默认焦点且不提供替代样式。
- **语义可读**：图标按钮需提供可读名称（如 `aria-label`）；表单控件需有可读标签。
- **状态冗余**：状态变化不得仅依赖颜色，需配合文案、图标或形状差异。
